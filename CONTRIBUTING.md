# Contributing to PyTermStructure

Thank you for considering contributing to PyTermStructure!

## GNU GPLv3 License

**Important**: All contributions must be compatible with GNU GPLv3.

By contributing, you agree that your contributions will be licensed under the same GNU GPLv3 license.

## Reporting Bugs

Open an issue on GitHub with:
- Python version
- Operating system
- Minimal reproducible example
- Expected vs actual behavior

## Suggesting Features

Open an issue tagged "enhancement" with:
- Use case description
- Proposed API
- Why it'"'"'s useful

## Pull Requests

1. Fork the repository
2. Create feature branch: `git checkout -b feature/my-feature`
3. Make changes with tests
4. Run tests: `pytest`
5. Commit: `git commit -m "Add feature X"`
6. Push: `git push origin feature/my-feature`
7. Open Pull Request

### Code Style

- Follow PEP 8
- Add type hints
- Document with docstrings (NumPy style)
- Add tests for new features

### Priority Areas

1. **Accuracy improvement** (target: <5 bps)
2. **Full pseudoinverse** (cash flow matrix)
3. **Lorimier optimization**
4. **Day-count conventions**
5. **Business day calendars**
